How to install in Blogger

  1. Back up your current theme first In Blogger admin: Theme → ▼ next to "Customize" → Backup. Save the .xml file somewhere safe — that's your rollback if anything looks wrong.
  2. Restore the new theme Same menu: Theme → ▼ → Restore → upload coding-mayhem-workshop.xml. Blogger will validate and apply it. Your posts, pages, comments, and labels are untouched.
  3. Add widgets via Layout Go to Layout. The header, primary nav, and footer are pre-wired. The sidebar holds About / Tags / Recent & popular / Archive — you can reorder, add, or remove widgets there.
  4. Make sure posts have a featured image The homepage hero and post cards pull from each post's featured image (Blogger sets it from the first image in the post, or you can set it explicitly in the post editor sidebar). Posts without one show a striped fallback that reads NO COVER.
  5. (Optional) Tweak the accent color Theme → Customize → Advanced → Accent color. Default teal is #0f7a6c (light) / #5fd1be (dark).

What's included

Theme system
Light / dark toggle in the header. Remembers your choice in localStorage and respects system preference on first visit. No flash on load.
Homepage
Hero featured post + 2-column card grid + sidebar with About / Tags / Popular / Archive widgets.
Post page
Sticky table-of-contents built from your H2/H3s, reading-progress bar, big author bio block, adjacent-reading cards pulled from RSS.
Code blocks
Decorated <pre> with traffic-light dots, language label, and one-tap copy. Add language-bash (etc.) on the <code> tag to label it.
Typography
Manrope for everything, JetBrains Mono for code, metadata, and tags. Loaded from Google Fonts with preconnect for fast rendering.
Search & nav
⌘-style search overlay (press / anywhere) that posts to Blogger's native search. Pages widget renders the top nav automatically.
Heads up — Blogger requires that widget IDs are preserved when you switch themes. The XML uses Header1, Blog1, Blog2, PageList1, Profile1, Label1, PopularPosts1, BlogArchive1, Attribution1. If your current theme has different widget IDs (e.g. HTML4, Linklist2), Blogger will warn you during restore and either remap or ask you to delete the orphans. That's expected — accept the prompt. Your post content lives in the Blog1 widget which always exists and is never touched.
